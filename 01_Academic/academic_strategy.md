# 3. Academic Strategy (UniCT L-8)

The current objective is to complete the Bachelor's degree while maintaining a strong foundation in core computer engineering subjects.

## 3.1 Course Categorization

Courses are prioritized based on their relevance to the long-term professional goal:

**Priority 1: Core Technical Foundation**

Direct prerequisites for the technical portfolio and target roles in embedded systems / Edge AI.

- **Computer Architecture** — understanding memory hierarchies, pipelines, and instruction sets is non-negotiable for writing efficient bare-metal firmware
- **Operating Systems** — concepts map directly to RTOS (FreeRTOS, Zephyr) and embedded Linux; scheduling, IPC, and memory management are daily tools
- **Programming (C/C++)** — the primary language of embedded systems; mastery is the hiring baseline for every target role
- **Machine Learning** — academic foundation for the on-device inference component; bridges the portfolio's Edge AI layer

**Priority 2: Supporting Engineering Subjects**

Reinforces the core stack and broadens hardware-level competence without being directly portfolio-critical.

- **Electronics** — essential for hardware bring-up, reading schematics, and debugging signal-level issues on real boards
- **Computer Networks** — relevant for IoT/Edge networking layers and protocol understanding (TCP/IP, sockets used in Linux phase)
- **Algorithms and Data Structures** — underpins efficient preprocessing pipelines and model feature extraction; necessary for technical interviews
- **Electrical Engineering** — provides physical intuition for power budgets, signal integrity, and sensor interfacing
- **Object-Oriented Programming** — not for Java per se, but for the OOP paradigm: class design, abstraction, and driver layering patterns used in C++ embedded code

**Priority 3: High-Level Concepts**

Valuable for professional breadth but not directly tied to the portfolio deliverables.

- **Databases** — limited direct use; relevant only for logging/telemetry in Linux-based phases
- **Software Engineering** — process and tooling knowledge (version control, CI, documentation) that improves portfolio quality rather than technical depth
- **Security** — tangentially relevant via the CAN bus IDS research project; not a primary focus
- **Automatic Control** — useful background for sensor fusion and actuator feedback loops; low immediate priority

**Priority 4: General Engineering Requirements**

Degree prerequisites. Completion is the goal; deep mastery is not required for the target career path.

- **Mathematics (Analysis, Geometry)** — required degree credits; foundational but not actively applied in portfolio work
- **Physics** — degree requirement; physical intuition useful but not a differentiator
- **Signal Theory** — relevant to sensor signal processing at a conceptual level; depth not needed beyond exam pass
- **Digital Communications** — degree requirement; overlaps marginally with CAN/network protocol work
  _Strategy: Prioritize completion and credit acquisition over exhaustive mastery in this category._
